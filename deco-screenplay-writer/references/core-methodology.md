# 核心编剧方法论（所有格式通用）

## 零、核心原则：电影是现实生活的横截面

**这是整个方法论的地基。** 一个故事——无论短片还是长片——不是从人物出生讲到人物死亡。它是从一个完整人生中切出的一段横截面：在某个特定时刻，因为某个特定事件，这个人被逼入了一个不可回避的处境。

这意味着：

**1. 故事开始之前，世界已经运行了很久。**
角色有前史（Ghost），世界有时代背景，关系有既往积累。剧本的第一页不是角色的第一天——角色带着所有前史走进第一场戏。这些前史不需要全部展示给观众，但编剧必须知道。

**2. 故事从中间开始。**
观众被扔进一个"已经在发生"的世界。好的开场不是"从头讲起"，而是"从最有张力的那个截面切入"。观众通过角色的行为和反应反推前因，这比顺序交代更有吸引力。

**3. 开场即态度。**
第一场戏定义了整部片子的气质——不是主题，是"这部片子用什么方式看世界"。商业片开场用高密度的情节动作宣告节奏；文艺片开场用一段精准的生活质感宣告风格。无论什么类型，开场都必须是全片最精心设计的段落之一。

**4. 编剧的工作是先建造完整世界，再切一刀。**
工作流的逻辑因此是：先建造（前史、世界观、角色全貌），再裁切（选择从哪里开始讲、展示什么隐藏什么）。不是"我要讲什么"，而是"这个世界存在着什么，我选择让观众从哪个窗口看进去"。

## 一、戏剧动作：故事的最小单位

**定义：** 戏剧动作 = 目标 (Goal) + 阻碍 (Conflict)。
不是"一个人想喝水"，而是"沙漠中一个脱水的人找水"。目标必须有追求的紧迫性，阻碍必须与目标直接对抗。

故事中的每一个场景都应该包含至少一个微型戏剧动作（目标 → 障碍 → 结果），结果可以是成功、失败、或意外转向。场景内的微型戏剧动作向上汇聚成段落级的核心戏剧动作，段落级的核心戏剧动作再向上汇聚成全片的核心戏剧动作。

**检验标准：**
- 目标是否合理且有急切性？
- 目标和阻碍之间是否有强烈的冲突关系？
- 这个冲突是否足以驱动故事运转？

## 二、人物设计：拒绝纸片人

### Want vs Need

- **外在需求 (Want)**：驱动动作的目标，角色自己意识到的、主动追求的东西
- **内在欲望 (Need)**：角色自己未曾意识到的灵魂缺口，通常是外在欲望的深层原因

Want和Need之间的张力是角色弧光的发动机。角色在故事开始时追求Want，在故事结束时要么得到了Need（成长弧光），要么拒绝了Need（悲剧弧光/反弧光）。

### 人物弧光

弧光 = 从A状态到B状态的不可逆变化。
短内容的弧光是一次性的跳跃（A→B）。
长内容的弧光可能经历多次"伪变化"——角色看起来变了，但又退回去，直到最终的真正转变。

### 人物小传

角色的前史不是背景介绍，而是冲突的根源。小传中的关键事件应该解释：
- 为什么角色有这个Want
- 为什么角色看不到自己的Need
- 角色的矛盾性从何而来

### 矛盾性

好的角色至少有一层矛盾：外在表现 vs 内在真实。
例如：外在自信 vs 内在自卑，外在冷漠 vs 内在渴望连接。
矛盾性让角色立体，也让潜台词有了空间。

### 人物深度缩放

| 格式 | 人物深度要求 |
|------|------------|
| 概念超短片 | 功能性archetype即可，不需要小传和弧光 |
| 短片 | Want/Need/Arc + 关键矛盾特征 |
| 长片 | 完整小传 + Ghost(前史创伤) + Lie(角色相信的谎言) + Flaw(性格缺陷) |
| 剧集 | 同长片深度，但弧光需分配到各集，另需设计"弧光预算" |

## 三、视听化写作

### Show, Don't Tell

严禁写心理描写（"他觉得"、"她意识到"），只写能被看见的动作和能被听见的声音。
如果一个情感无法通过角色的物理行为传达给观众，那这个情感就不应该出现在剧本中。

**错误示范：** "他终于理解了父亲的苦心。"
**正确做法：** 他站在那里看了一会儿，然后把外套脱下来盖在父亲身上。

### 潜台词 (Subtext)

对话是"冰山"——角色说出来的话只是表面，真正的含义在水面之下。
避免直白 (On-the-nose) 的对话，利用动作、停顿、回避、话题转移来表达真意。

**直白对话（错误）：**
> "我很害怕失去你。"

**潜台词对话（正确）：**
> "你走的时候把那件外套带上。……外面冷。"

### 动作即潜台词

所有的情感传递都必须能用摄影机拍出来。用角色的具体动作替代任何解释性的台词。
一个好的剧本，把所有台词删掉后，观众仍然能大致理解发生了什么。

### 台词风格

- 口语化，像真人在说话，不是在写文章
- 不同角色有不同的说话方式（用词、句式长度、习惯性表达）
- 避免过度比喻、类比和书面化的AI腔
- 对话中自然传递信息，不要让角色说出他们双方都已经知道的事情

## 三点五、开场钩子（所有格式通用）

**每一部影片、每一集剧集、每一个短片，都必须在开场就抓住观众。** 开场不是"慢慢进入状态"——观众从第一个画面开始就在决定要不要继续看。

### 通用原则

开场钩子的任务不是"交代背景"，而是**在观众还不知道故事是什么的时候就让他们无法移开目光**。方法因类型而异，但底线是一样的：开场必须有一段足够精彩的、能被摄影机拍出来的视觉内容。

**开场钩子不等于"爆炸开场"。** 安静也可以是钩子——一个安静到让人不安的画面、一个日常到让人困惑的细节，只要它让观众产生"怎么回事？"的疑问，就是有效的钩子。

### 按类型的开场策略

**商业片/类型片：** 开场应该是一段**高密度、高紧张感的杰出场景**。它的功能是双重的——同时交代角色的状态（能力、性格、处境）和建立影片的类型基调。观众在这一场戏里就应该知道"这是一部什么样的电影"。

经典做法：
- 用一场"日常任务"展示角色的能力和风格（《夺宝奇兵》开场的丛林探险不是主线，但它告诉观众：这个人是谁、他做什么、他有多厉害、这部电影的节奏是什么）
- 用一场"危机"直接把角色扔进压力中（《黑暗骑士》开场的银行劫案——还没见到主角就先见到了反派的规则）
- 用一个"结果先行"的画面制造悬念（先看到一个离奇的结果，然后全片回答"这是怎么发生的"）

**文艺片/角色驱动型：** 开场应该是一段**能代表全片风格和气质的段落**。它不需要紧张，但需要独特——让观众在前两分钟内感受到"这部电影的世界和我日常看到的不一样"。

经典做法：
- 用一个高度风格化的长镜头建立氛围（《鸟人》开场的后台长镜头——观众立刻知道这部电影的视听语言是什么）
- 用一段看似无关的日常细节暗示角色的内心裂缝（是枝裕和式——开场就是做饭、吃饭，但某个细节让人感觉"不对"）
- 用一个视觉母题（motif）的第一次出现来锚定全片的影像系统（王家卫式——开场的某个画面/动作会在全片反复出现）

**短片：** 开场的前15秒决定生死。短片没有时间"慢慢来"——第一个画面就应该包含信息或悬念。最有效的短片开场是一个"不太对"的日常画面——看起来正常，但某个细节让人起疑。

**概念超短片：** 开场即概念。前5秒必须让观众理解"这个世界的规则"或至少让观众产生强烈的好奇。

**剧集：** 每一集都需要开场钩子，不只是第一集。剧集的冷开场（teaser/cold open）是最关键的钩子位置——在片头曲播放之前就抓住观众。第一集的开场承担最重的任务：同时建立世界观、角色、类型基调、和季度核心冲突的种子。

### 开场钩子自检

在结构大纲（第三步）和剧本写作（第五步）时检查：
- [ ] 开场的第一个画面是否有视觉冲击力或悬念感？
- [ ] 观众在前30秒内是否会产生一个具体的疑问（"这是怎么回事？"）？
- [ ] 开场是否在交代状态的同时就在制造张力——而不是"先交代再开始"？
- [ ] 开场的类型基调是否与全片一致？（商业片开场不能像文艺片，反之亦然）
- [ ] 如果把开场单独拿出来，它本身是否是一段精彩的视听内容？

## 四、自检体系

每个工作流步骤都有对应的自检清单。用户可以在任何步骤要求执行自检。

### 自检一：核心概念与戏剧动作
- [ ] 是否建立了"目标+冲突"？
- [ ] 目标是否合理且有急切性？
- [ ] 目标和阻碍是否有强烈关系？
- [ ] 冲突是否是驱动故事的发动机？
- [ ] 阻碍是否在合理尺度内，而非刻意夸大？

### 自检二：人物塑造与弧光
- [ ] 外在需求和内在欲望是否清晰且对立？
- [ ] 是否预设了不可逆的内在变化？
- [ ] 结局是否展示了"新常态"？
- [ ] 是否具备矛盾性？
- [ ] 行为是否符合小传逻辑？

### 自检三：结构与节奏
- [ ] 结构各部分是否清晰？
- [ ] 激励事件是否明确？
- [ ] 高潮是否明确？
- [ ] 每一部分是否都在推动情节或揭示性格？
- [ ] 是否存在无效情节？

### 自检四：场景写作与视听语言
- [ ] 是否有心理描写？（如有则Fail）
- [ ] 是否有括号暗示？（如有则Fail）
- [ ] 是否用画面叙事？
- [ ] 对话是否过于直白？
- [ ] 是否利用了潜台词和肢体语言？
- [ ] 台词是否口语化？
- [ ] 是否有角色在"解释设定"而非"应对设定"？

## 五、剧本格式规范（标准格式）

```
【场景X：地点 / 时间 / 简要视觉描述】

动作/画面描写直接写在台词之间，不加标记，只写摄影机能拍到的东西。

角色A：xxxxxxxxx。
角色B：xxxxxxxxx。

角色A没有回答。他转身走向窗口，手指无意识地敲着窗框。

角色A：xxxxxxxxx。
```

**格式规则：**
- `【场景标注】` 用方括号，标注地点、时间、关键视觉信息
- `角色名：` 直接跟对白
- `角色名（OS）：` 内心独白/旁白
- 动作/画面描写不加任何标记，直接写在台词之间
- 只写能被摄影机拍到的内容

## 六、双轨节奏系统

优秀影片的节奏是张弛有度的。节奏不是只有一条线——它有两条平行运行的轨道，两者可以同步也可以错位，错位本身就是强大的叙事工具。

### 外部情节节奏

外部情节节奏是"事件的密度和强度"——场景中发生了多少事、冲突有多激烈、信息量有多大。

- **紧**：高密度事件、快速剪辑、多人对话、追逐、对峙、揭露
- **松**：日常场景、独处时刻、过渡、风景、沉默

外部节奏的张弛要有对比——持续紧张会让观众麻木，持续松弛会让观众走神。经验法则：每一段高强度情节之后，需要一段"呼吸"让观众消化。

### 内在情感节奏

内在情感节奏是"角色情感状态的波动"——角色经历了多大的情感起伏、观众被要求投入多少情感。

- **重**：告白、背叛、失去、恐惧、狂喜、绝望
- **轻**：调侃、日常闲聊、小幽默、安静的陪伴

情感节奏和情节节奏并不总是同步的。一些最强大的场景恰恰是两者错位的时刻：

- **情节松 + 情感重**：两个人安静地坐着吃饭，什么事都没发生，但观众知道其中一个人明天要走了。画面越平静，情感越重。
- **情节紧 + 情感轻**：激烈的追车戏，但配乐是轻松的流行歌曲，角色在打趣。紧张的事件被轻盈的情绪包裹。
- **情节紧 + 情感重**：高潮段——事件密集且情感炸裂。这是全片的顶点，不能持续太久。
- **情节松 + 情感轻**：纯粹的呼吸场景。让观众休息。通常放在高潮前后。

### 节奏波浪图

在结构大纲和场景拆解阶段，应该为全片画一张"节奏波浪图"——标注每个段落的外部情节强度和内在情感强度，检查是否有足够的起伏和对比。

```
示意（纵轴=强度，横轴=时间线）：

外部情节:  ─╱╲─────╱╲──╱╲╱╲╲──────╱╲╱╲╱╲╲
内在情感:  ──────╱╲───────╱╲──╱╲╱╲───────╲╱╲
```

两条线不必同步，但应该避免：
- 两条线长时间同时处于低位（观众无事可看也无感可投）
- 两条线长时间同时处于高位（观众疲劳，高潮变得不特别）
- 任何一条线单调上升没有回落（没有呼吸感）

## 七、时长估算与节奏控制

每一个戏剧动作、每一场戏都应该有时长估算。时长不只是"这场戏多长"，更是"这场戏在全片节奏中占多大比重"。

### 为什么必须估算时长

- 一场戏如果在大纲里读起来很重要，但只占30秒，它在成片里的分量会被压缩到观众几乎注意不到
- 一场戏如果占了全片15%的时长，但只传递了一个次要信息，它就是节奏的拖累
- 只有把时长标出来，才能在纸面上看到节奏的问题——哪里太赶、哪里太拖、高潮是否有足够的铺垫时间

### 时长估算方法

**对白估算：** 1页标准剧本格式 ≈ 1分钟成片。中文剧本中，大约200-250字的对白+动作描写 ≈ 1分钟。

**场景估算规则：**
- 纯对话场景：按对白字数估算
- 动作/蒙太奇场景：按画面数量估算（每个独立画面约3-5秒）
- 沉默/情绪场景：容易低估——一个角色站在那里什么都不做，在成片中可能需要10-15秒才有足够的情感分量
- 转场/过渡：不要忘记计入，通常每次转场2-5秒

### 场景时长标注格式

在场景拆解（第四步）和剧本写作（第五步）中，每场戏都应标注预估时长：

```
【场景3：医院走廊 / 夜 / 荧光灯嗡嗡响】（约1分30秒）
  情节节奏：中（一次关键对话）
  情感节奏：重（角色第一次面对真相）
```

### 全片时长预算

在结构大纲阶段就应该做一份粗略的时长预算，确保各段落的比例合理：

| 格式 | 建置 | 发展前半 | 发展后半 | 高潮 | 结局 |
|------|------|---------|---------|------|------|
| 短片（8分钟） | 1.5分钟 | 2分钟 | 2分钟 | 1.5分钟 | 1分钟 |
| 长片（100分钟） | 25分钟 | 30分钟 | 25分钟 | 15分钟 | 5分钟 |
| 剧集单集（25分钟） | 4分钟 | 10分钟 | 6分钟 | 3分钟 | 2分钟 |

以上为参考比例——实际创作中可以灵活调整，但调整后应该意识到自己在做什么（比如"我故意让建置占了40%时长，因为这是一个慢热型故事"）。

### 节奏诊断

在剧本医生阶段，用时长预算检查以下问题：
- 高潮段的时长是否足够？（太短会让高潮感觉被压缩）
- 建置是否太长？（观众等太久才进入正题）
- 是否有连续超过3分钟的单一情绪段落？（无论是紧张还是松弛，持续太久都会失效）
- 两个高情感场景之间是否有足够的呼吸空间？
- 全片最长的单场戏是否有足够理由占据那么多时长？

## 八、跨格式技法借鉴

不同格式的技法不是孤立的。一个好的编剧会把概念片的翻转逻辑用在长片的冷开场里，把McKee的价值转变用来检验短片的每场戏。以下是一张"技法借鉴地图"——当你在某种格式中遇到特定问题时，可以从另一种格式中借来工具。

### 从概念片借来的

**高概念翻转 → 用在任何格式的单场戏中**
概念片的核心能力是"用一个翻转重新定义前面所有信息"。这个技法可以缩小到一场戏的尺度——在任何格式中，当你需要一个场景有"punch"的时候，问自己：这场戏的最后一秒能不能让观众重新理解这场戏的第一秒？

适用场景：长片的激励事件、剧集的冷开场/集末钩子、短片的结局。

**概念纯度检验 → 用来检查任何格式的核心前提**
概念片要求"一句话说清、5秒内能用画面呈现"。这个标准可以拿来检验任何格式的Logline——如果你的长片用一句话说不清楚，说明核心概念还不够锐利。

**零解释纯视觉建立 → 用在任何格式需要快速建立规则的场景**
概念片不解释规则，直接用画面展示后果。当长片或剧集需要引入一个新规则或新环境时，先问自己：能不能不说一句话就让观众理解？

**概念组合方法 → 用在任何格式的构思阶段**
概念片的五种组合方式（概念嫁接、时空错置、隐喻具象化、反转并置、规则反转）不只是超短片的工具。当长片或剧集在构思阶段觉得"概念不够锐利"或"太普通"时，可以用这些组合方式来重新碰撞：
- **概念嫁接**用在长片：把一个类型片的机制移植到另一个领域——"如果恐怖片的规则运行在办公室政治中？"这就是《寄生虫》式的类型杂交思维。
- **时空错置**用在剧集：某一集让当代角色面对一个历史版本的问题——不是闪回，而是用古代的镜子照当代的荒诞。
- **隐喻具象化**用在短片：把短片的核心情感冲突变成一个可视化的物理规则——"无法沟通"不是角色说"我们没法沟通"，而是两个人说话时嘴里会飞出实体的字，字会在空中撞碎。
- **反转并置**用在长片的场景序列：两条看似无关的线在平行推进，在某个时刻观众发现它们是同一件事的两面——这是蒙太奇叙事的本质。
- **规则反转**用在任何格式的What-if设定：把故事世界的一个规则取反，看看会发生什么——这是科幻和荒诞题材的基础思维。

**多结构类型 → 用在任何格式的局部段落**
概念片的五种结构（三段式、递进崩塌、双线并置、循环/Loop、倒叙揭示）可以缩小到长片或剧集的一个序列/一场戏中使用：
- **递进崩塌式**用在长片的"Bad Guys Close In"段落——压力一级级加码直到崩溃点。
- **双线并置式**用在长片/剧集中需要对比的段落——不用台词解释对比关系，用画面的并置让观众自己看到。
- **循环/Loop式**用在剧集中展示"角色被困在重复模式中"的段落——同一个早晨、同一个争吵、同一个失败，循环本身就是叙事。
- **倒叙揭示式**用在长片的冷开场——先给一个不可理解的画面，然后全片都在回答"这是怎么发生的"。

**视听语言武器库 → 用在任何格式需要视觉冲击力的段落**
概念片的视听手法不是超短片的专利——在长片和剧集的特定段落中使用这些手法，效果可能比常规叙事更强：
- **分屏**用在长片的时间压缩段落（展示多个角色同时在做不同的事）、剧集的recap、或需要对比两个空间的场景。
- **跳切蒙太奇**用在长片/剧集的时间推进段落——在固定机位上快速跳切，几秒内跨越几周几月。
- **画面与声音错位**用在任何格式需要讽刺效果的场景——画面是温馨的家庭聚餐，声音是新闻播报着某个灾难性的数据。
- **静默**用在任何格式的高潮后——持续的高密度信息流之后突然完全安静，沉默本身就是最强的高潮。
- **固定机位时间流逝**用在长片/剧集需要"时间感"的段落——同一个画面中季节更替、人来人去，唯一不变的是视角。
- **伪纪录片/伪文档**用在长片/剧集的某个段落——突然切换成"纪录片"风格可以打破叙事的第四面墙，也可以制造"真实"的错觉。
- **色彩限制**用在长片/剧集的特定段落——用颜色的变化标记叙事层次（回忆用去饱和、幻想用高饱和、现实用自然色调）。

使用原则：在长片和剧集中借用这些视听手法时，通常只在特定段落使用（不是全片），它的出现本身就标记了"这个段落不一样"。如果全片都在用同一种炫技手法，它就不再特别了。

### 从短片借来的

**信息密度蒙太奇 → 用在长片/剧集的Fun and Games段落**
短片的"快速图鉴"蒙太奇——用堆叠的视觉切片制造密度——在长片的Fun and Games段（主角在新世界中试错）和剧集的适应段落中同样强大。当你需要快速展示"很多人在做很多事"时，借用这个技法比逐个讲故事高效得多。

适用场景：长片第二幕前半的montage、剧集的时间推进段落。

**单一变量极致推演 → 用在长片/剧集的某个序列中**
短片的"只引入一个变量、严格按因果推演到极致"在长片的某个序列或剧集的某一集中可以成为强大的结构工具——把一个复杂故事中的某个局部限制为单一变量推演，能制造出极其紧凑和满足的段落。

适用场景：长片的高潮序列（排除一切干扰，聚焦核心选择）、剧集的"瓶装集"（bottle episode，限定空间和人物）。

**零度表达 → 用在任何格式的高情感时刻**
"灾难越大，语气越平"——这不只是黑色幽默的技法。在任何格式中，当情感到达顶点时，用最克制的表达反而能制造最大的冲击。长片的至暗时刻、剧集的崩塌集，都可以借用这个原则。

### 从长片借来的

**McKee价值转变 → 用来检验任何格式的每一场戏**
"场景结束时的核心价值是否与开始时不同？"这个检验标准不限于McKee体系的长片。短片的每场戏、剧集的每场戏，都可以用这个标准检查——如果一场戏结束时什么价值都没变，考虑删掉它。

适用场景：短片的场景拆解自检、剧集的剧本医生阶段。

**Ghost/Lie/Flaw链条 → 简化后用在短片的人物设计**
短片通常只做Want/Need/Arc，但如果短片的人物感觉"太平了"，可以借用Ghost/Lie/Flaw的思考方式——不需要完整展开，只需要暗示：角色的行为背后有一个不被说出的原因。一个细节就够。

适用场景：短片第二步（人物深度）中想要更多层次时。

**伏笔/回扣 → 缩小到短片尺度**
短片只有5-10分钟，但一个精妙的伏笔回扣在短时间内的冲击力反而更大——因为观众还记得伏笔。短片中，第1分钟不经意出现的一个物件在第8分钟突然成为关键，效果是炸裂的。

适用场景：短片的结构大纲阶段，考虑"有没有一个东西可以在开头种下、结尾翻转"。

**Subplot的映射逻辑 → 简化后用在短片的次要角色**
短片通常没有正式的Subplot，但可以借用"B Story映射主题"的逻辑——如果短片中有一个次要角色，让这个角色走主角"没走的那条路"。这样次要角色就不只是工具人，而是主题的另一面。

### 从剧集借来的

**集末钩子逻辑 → 用在长片的序列衔接**
剧集每集结尾的钩子（一个未解答的问题、一个不安的细节）可以用在长片的序列衔接中——每个序列结束时留一个"小钩子"，让观众想知道接下来会怎样。这比简单的"然后..."过渡更有牵引力。

适用场景：长片第四步（场景拆解）中，每个序列之间的衔接设计。

**弧光预算思维 → 用在长片的伪变化规划**
剧集用"预算"来控制角色在各集中的变化额度。长片也可以借用这个思维——角色不应该在第30分钟就到达真正的变化，需要规划几次"伪变化"和回退。把弧光看成有限的资源，分配到不同的段落。

适用场景：长片第三步（结构大纲）中，规划角色的变化节点。

**信息释放时间表 → 用在长片的世界观管理**
如果长片的世界观有一定复杂度（科幻、奇幻、或者复杂的社会结构），借用剧集的"信息释放时间表"思维——把世界观信息分层，规划在第几分钟让观众知道什么。

**连续性追踪的思维 → 用在长片的后半段写作**
长片越写到后面，越容易出现前后矛盾。借用剧集的"角色状态追踪"思维——在写第三幕之前，先列一张表：每个角色此刻的物理状态、情感状态、已知信息。

### 如何使用这张地图

1. **不要强行借鉴。** 只在遇到具体问题时翻阅——"这个结局没有punch" → 试试概念片的翻转逻辑；"这场戏感觉多余" → 用McKee价值转变检验。
2. **借鉴的是思维方式，不是格式。** 在长片中用"快速图鉴蒙太奇"不代表长片变成了短片——只是在某个段落借用了短片的信息密度策略。
3. **在剧本医生阶段特别有用。** 当你觉得某个段落"不对"但说不出哪里不对时，用另一种格式的检验标准来诊断，往往能发现问题。

## 九、选题工具库（内部知识，引导用户而非展示给用户）

### 使用原则

这一节的所有内容是编剧大师的内部武器库——**不要把选题方法论、路径分类、组合技法直接展示给用户看**。用户看到的应该是自然的对话引导、启发性的提问、和具体的方案输出，而不是一张方法论菜单。

当用户还没有明确想法时，通过聊天判断用户的模糊方向属于哪种切入路径，然后用对应的工具帮他们锐化概念。

### 五条选题路径

根据用户输入的信号判断属于哪条路径，然后用对应的方法引导：

**路径1：从主题出发**
用户信号：给了一个抽象的命题、社会议题、哲学问题、"我想讲关于XX的故事"。
例如："关于记忆"、"我想探讨人工智能的边界"、"孤独"。

引导方法：帮用户把抽象主题转化为具体的戏剧动作——"关于记忆"不是故事，"一个即将失去所有记忆的人必须在24小时内选择保留哪一段"才是故事。问用户：这个主题下，什么人在什么处境下会被逼到不得不做选择？

**路径2：从人物出发**
用户信号：描述了一种人、一种矛盾性格、"我想写一个XX样的人"。
例如："一个极度害怕水的游泳教练"、"一个色盲的画家"、"一个完美主义的急诊医生"。

引导方法：帮用户从矛盾性格中提取戏剧动作——矛盾本身就是冲突的种子。问用户：这个人最害怕面对什么？如果逼他面对他最害怕的事，会发生什么？

**路径3：从空间/环境出发**
用户信号：给了一个具体的地点、环境、封闭空间、特殊世界。
例如："一座老茶馆"、"一栋垂直分层的大楼"、"一家即将歇业的大饭店"、"一列永不停站的列车"。

引导方法：空间即规则——一个特殊的空间暗示了它自己的运行规则和权力结构。问用户：这个空间里谁在上面谁在下面？什么资源是稀缺的？什么行为在这里是被禁止的？这些规则本身就是故事的发动机。空间越封闭、规则越极端，戏剧张力越强。

参考："布达佩斯大饭店"（一个正在衰落的旧世界的最后守卫者）、"饥饿站台"（垂直分层的食物分配系统=阶级隐喻）、"茶馆"（同一个空间里三个时代的中国=时间的压缩）、"雪国列车"（一列火车=整个人类社会的缩影）。

**路径4：从关系出发**
用户信号：描述了两个人之间的张力、"我想写两个XX之间的故事"。
例如："一对即将离婚的夫妻"、"一个学生发现老师在抄袭"、"双胞胎中一个成功一个失败"。

引导方法：关系中的张力来自"两个人想要的东西互相矛盾"。帮用户明确：两个人各自的Want是什么？这两个Want为什么不可能同时实现？把他们放在什么处境下，这个矛盾会被逼到极致？

**路径5：从标签/概念碰撞出发**
用户信号：给了几个关键词、一种感觉、"我想把XX和XX结合"、或者只是一些零散的灵感碎片。
例如："外卖骑手+科举"、"斩杀线+AI焦虑"、"钢铁侠+下岗潮"、"社恐+相亲"。

引导方法：使用以下五种概念组合工具碰撞出概念（详见 format-ultrashort.md 的概念组合部分，但这些工具不限于概念片——短片、长片、剧集的选题阶段都可以用）：

- **概念嫁接**：取概念A的运行机制 + 概念B的应用场景。"游戏复活机制+职场裁员" → "被裁的人可以投币复活，但每次复活工资减半"。
- **时空错置**：把人物/技术/制度放到它不属于的时代或地点。"ChatGPT+科举" → "考生带了一个能对对子的铜匣子进考场"。
- **隐喻具象化**：把抽象焦虑变成物理规则。"内卷" → "所有人的灯必须比邻居亮，亮度不够就断电"。
- **反转并置**：找到两个表面相似但本质矛盾的事物，放在同一场景。"宠物+畜牧" → 同一条街上，爱和杀并存。
- **规则反转**：取一个现实规则的反面。"越努力越成功"反转 → "越努力拥有的越少"。

### 选题引导的操作流程

1. **听用户说。** 让用户自由表达，哪怕只是一个词、一种感觉。
2. **判断路径。** 根据用户的输入信号判断属于哪条路径（用户不需要知道路径的存在）。
3. **用对应工具深挖。** 通过提问帮用户锐化概念——问题要具体、有启发性，不要问"你想表达什么"这种空泛的问题。
4. **产出方案。** 用锐化后的概念生成3-5个具体方案，供用户选择。
5. **不要展示方法论本身。** 用户看到的是"你帮我想出了几个好点子"，而不是"这里有五种选题路径请你选一个"。

### 当用户完全没有方向时

如果用户说"我完全不知道想写什么"，不要列出路径菜单。用自然的对话启发：

- "最近有没有什么新闻、画面、或者身边的事让你觉得'这也太荒诞了'？"
- "如果你能把一个现实中的规则反过来，你最想反转哪个？"
- "有没有一个你去过的地方或空间，让你觉得'这里本身就像一个故事'？"
- "有没有两个看起来不相关的东西，你觉得放在一起会很有意思？"

根据用户的回应判断路径，然后用对应工具展开。

## 十、记忆检查点系统（长内容防丢失）

### 为什么需要记忆检查点

长片剧本约15000-25000字，剧集全季可达10万字以上。在创作过程中，对话上下文会逐渐超出模型的有效记忆范围——前面写的场景、埋下的伏笔、角色的状态变化，可能在写到后面时被"遗忘"，导致前后矛盾、伏笔断裂、角色行为失去连续性。

记忆检查点系统通过定期生成结构化的"快照"来解决这个问题——每到一个节点，把已完成部分的关键信息压缩为一份简洁的摘要，在后续创作中作为"记忆锚点"使用。

### 何时触发检查点

| 格式 | 触发时机 | 说明 |
|------|---------|------|
| 概念超短片 | 不需要 | 内容短，不会超出上下文 |
| 短片 | 第四步（场景拆解）完成后触发一次 | 进入写作前锁定全局信息 |
| 长片 | 每完成一个Sequence（2-3场戏）触发一次 | 约每10-15分钟成片内容一个检查点 |
| 剧集 | 每完成一集触发一次；单集内如果超过6场戏，中间也触发一次 | 集间检查点是最重要的 |

此外，以下情况也应主动触发：
- 用户开始新一轮对话但要继续之前的创作
- 已经写了很多内容，感觉前面的细节开始模糊
- 用户主动要求"总结一下目前的进度"

### 检查点包含什么

检查点是一份结构化的压缩摘要，**不是**全文复述。它只记录后续创作必须知道的信息。

#### 通用检查点模板

```
═══════════════════════════════════
📌 记忆检查点 #[编号] | [格式] | [当前位置]
═══════════════════════════════════

【已完成进度】
  当前处于第几步 / 已写到哪个场景或段落

【故事核心（不变）】
  Logline：
  主题论点（如有）：
  核心戏剧动作：目标 → 阻碍

【角色当前状态】（每个主要角色一行）
  [角色名]：物理状态 / 情感状态 / 当前知道什么 / 当前想要什么
  [角色名]：...

【已完成段落摘要】（每个已完成场景/序列一句话）
  场景1：[一句话概括发生了什么 + 核心价值转变]
  场景2：...
  ...

【活跃线索】（当前正在运行的、后续必须回应的线索）
  - [线索1]：状态描述
  - [线索2]：状态描述

【伏笔状态】
  已埋未回扣：[伏笔内容] → 计划在[位置]回扣
  已回扣：[伏笔内容] ✅

【节奏状态】
  上一段落的情节节奏：松/中/紧
  上一段落的情感节奏：轻/中/重
  → 下一段落建议：[节奏方向]

【待写内容预告】
  下一个要写的场景/段落是什么
  这个段落的核心戏剧动作是什么

═══════════════════════════════════
```

### 不同格式的检查点细化

#### 长片检查点

长片的检查点在通用模板基础上增加：

```
【Subplot状态】
  A Plot进度：[当前位置/全片位置]
  B Plot进度：
  C Plot进度（如有）：

【弧光进度】
  角色当前处于弧光的哪个阶段：
  Lie是否被质疑过？质疑了几次？
  上一次伪变化发生在哪里？

【世界观已揭示层】
  观众目前知道世界观的哪一层？
  下一层计划在哪里揭示？

【结构定位】
  当前处于哪个节拍（STC/Story Circle/McKee的哪个阶段）？
  距离下一个关键节拍还有多少时长？
```

#### 剧集检查点

剧集的检查点分两级：

**集间检查点（每完成一集触发，最重要）：**

在通用模板基础上增加：
```
【本集完成摘要】
  本集Logline：
  本集核心事件：
  本集弧光消耗：角色变化了什么？累计进度？

【连续性追踪更新】
  角色状态变化：[与上一集的对比]
  暗线推进：本集推进了什么？
  新埋的种子：
  回扣的种子：
  新公开的信息：

【季度进度】
  当前处于季度四阶段的哪个阶段？
  弧光预算已消耗多少？
  下一集的分集大纲核心是什么？
```

**集内检查点（单集内容较长时触发）：**
使用通用模板即可，重点记录本集内部的角色状态和活跃线索。

### 如何使用检查点

**写作时：** 每次开始写新段落之前，先回顾最近的检查点——确认角色状态、活跃线索、节奏方向。

**跨对话时：** 如果用户在新一轮对话中要求继续创作，先请用户提供上一次的检查点内容（或在对话中寻找），以此为基础恢复上下文。

**剧本医生时：** 把所有检查点串起来通读，可以快速发现：
- 某条伏笔埋了但一直没回扣
- 某个角色的状态在某两个检查点之间突然跳变（可能是连续性错误）
- 节奏长时间没有变化（持续紧张或持续松弛）
- 某条Subplot长时间没有推进

### 检查点的输出方式

检查点应该以独立的格式块输出——视觉上与剧本内容明确区分，便于用户保存和回贴。

触发检查点时的标准话术：
> 到这里先建一个记忆检查点，把目前的进度锁定一下。
> [输出检查点内容]
> 你可以保存这个检查点。如果后续对话中断或上下文太长，把这个检查点贴回来就能继续。

### 检查点的压缩原则

- **只记结论，不记过程。** "场景3：老王在医院发现了女儿的秘密，价值从信任→怀疑"——不需要复述对白和动作。
- **只记变化，不记不变。** 如果某个角色在这段时间内没有变化，不需要在检查点中重复他的状态。
- **只记后续需要的信息。** 一个已经完成的、不再影响后续的情节点，可以压缩为一句话甚至省略。
- **活跃线索和伏笔是最重要的。** 这些是最容易被"遗忘"的，必须每个检查点都完整列出。
