# Regional Anchor

Use `regional_anchor` to preserve the visible facts shared by assets that belong to one coherent world. It is reusable text, not an image asset.

## Define world truth

Derive only supported, generatable facts from the user's materials:

- region, era, climate, terrain, vegetation, air, and weather;
- social, class, occupation, and daily-life conditions;
- architecture, spatial scale, construction, and common materials;
- clothing, objects, infrastructure, grooming, footwear, and use traces;
- aging, wear, stains, humidity, dust, and practical-light sources;
- elements that would be regionally, historically, or socially wrong.

Do not turn the anchor into plot summary, biography, abstract theme, tourist imagery, invented brands, or generic prestige polish.

Use one anchor for each coherent visual world. Create another only when the materials establish a materially different geography, era, society, or material system.

## Draft efficiently

Write concise visible cues, not prompt prose. The anchor defines what can exist and how the world has aged; `style_aesthetic` defines how those facts are represented.

## Prevent conflict with style

- Keep regional materials, culture, climate, and lived condition legible under the chosen aesthetic.
- Do not let stylization erase the region, clean away required wear, modernize the era, or replace local construction with generic genre imagery.
- Do not use the anchor to prescribe a photographic or illustration finish that belongs in `style_aesthetic`.
- If the two directions conflict, identify the exact collision and give compatible alternatives for user confirmation.

After Preview approval, translate only the asset-relevant effects of this anchor into each production prompt.
